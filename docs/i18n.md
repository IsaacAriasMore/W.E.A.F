# Internacionalización ES/EN

Estado de Fase 8: navegación, footer, Home, FAQ, INIs, criaturas, Mapas & Bosses, directorio de servidores, planes, publicación, estados de Stripe y navegación/avisos legales usan claves compartidas ES/EN para su interfaz.

## API

El módulo `src/i18n/index.js` expone:

- `t(key, params)` para resolver texto y parámetros.
- `setLanguage(lang)` para cambiar y persistir el idioma.
- `getLanguage()` para leer el idioma activo.
- `initI18n()` para inicializar antes de renderizar la app.
- `subscribeLanguageChange(callback)` para componentes que necesiten reaccionar.
- `translateStaticDom(root)` para nodos con atributos `data-i18n`.

Los diccionarios están en `src/i18n/es.js` y `src/i18n/en.js`. Español es el fallback. La preferencia se guarda en `localStorage.weafLanguage`. Si no existe, se lee una vez `bossLanguage` para migrar el estado anterior de Mapas & Bosses.

## Agregar texto

1. Crear la clave en español.
2. Agregar la misma clave en inglés.
3. Usar `t('ruta.de.clave')` en HTML dinámico o `data-i18n="ruta.de.clave"` en contenido estático.
4. No concatenar oraciones traducibles. Usar parámetros como `{count}`.

## Reglas de cobertura

- Todo botón, error, empty state, encabezado y ayuda pública nueva debe nacer en ambos diccionarios.
- `tests/i18n.test.js` exige paridad de claves entre ES y EN.
- Los valores aportados por usuarios, nombres propios y datos del catálogo no se traducen en el navegador.
- Fechas visibles usan el locale activo cuando la pantalla ya fue migrada.
- No concatenar frases. Usar parámetros como `{count}`, `{title}` o `{date}`.

## Contenido editorial en Supabase

Las tablas actuales guardan un solo `title`, `description`, `notes` o `content`. Para traducir datos editoriales reales sin duplicar filas se necesita una migración futura, por ejemplo una tabla `content_translations` con entidad, ID, locale y campos traducibles. Esa decisión no se incluyó en Fase 8 para evitar un cambio de arquitectura y de RLS sin datos reales disponibles.

Hasta entonces:

- La interfaz cambia ES/EN.
- El contenido editorial local o remoto puede permanecer en el idioma de origen.
- No se debe fingir una traducción automática.

## Cobertura pendiente

- Contenido editorial de esas herramientas.
- Área privada y Admin, que continúan en español por prioridad operativa.
- Mensajes de servicios compartidos que hoy se generan en español antes de llegar a la vista.

La cobertura pendiente debe migrarse por superficie completa, incluyendo estados de carga, vacío, error y confirmación.
