# Internacionalización ES/EN

## API

El módulo `src/i18n/index.js` expone:

- `t(key, params)` para resolver texto y parámetros.
- `setLanguage(lang)` para cambiar y persistir el idioma.
- `getLanguage()` para leer el idioma activo.
- `initI18n()` para inicializar antes de renderizar la app.
- `subscribeLanguageChange(callback)` para componentes que necesiten reaccionar.
- `translateStaticDom(root)` para nodos con atributos `data-i18n`.

Los diccionarios están en `src/i18n/es.js` y `src/i18n/en.js`. Español es el fallback. La preferencia se guarda en `localStorage.weafLanguage`. Si no existe, se lee una vez `bossLanguage` para migrar el estado anterior de Mapas & Bosses.

## Agregar texto

1. Crear la clave en español.
2. Agregar la misma clave en inglés.
3. Usar `t('ruta.de.clave')` en HTML dinámico o `data-i18n="ruta.de.clave"` en contenido estático.
4. No concatenar oraciones traducibles. Usar parámetros como `{count}`.

La primera cobertura incluye navegación, footer, home, servidor público, formulario de publicación y facturación principal. Las zonas privada y admin pueden migrarse de forma incremental sin mantener un segundo sistema.
